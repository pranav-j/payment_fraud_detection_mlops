# Data

Datasets are **never committed to Git.** This directory and its subdirectories are gitignored except for `.gitkeep` markers and this README.

## Structure

- `raw/` — original PaySim CSV from Kaggle, untouched.
- `interim/` — partially processed dataframes (cached for notebook speed).
- `processed/` — final train/test splits ready for modeling.

## How to populate

```bash
bash scripts/download_data.sh
```

This pulls the [PaySim1 dataset by ealaxi from Kaggle](https://www.kaggle.com/datasets/ealaxi/paysim1) — about 470 MB, ~6.36 million rows of synthetic mobile money transactions.

## About PaySim

Synthetic dataset generated by Lopez-Rojas et al. to simulate mobile money transactions in an African economy. Released under CC BY-SA 4.0. The schema:

| Column | Type | Description |
|---|---|---|
| `step` | int | Time step in hours from simulation start (1..744 ≈ 1 month) |
| `type` | str | One of: CASH_IN, CASH_OUT, DEBIT, PAYMENT, TRANSFER |
| `amount` | float | Transaction amount in local currency |
| `nameOrig` | str | Originating account ID (drop for modeling — pure ID) |
| `oldbalanceOrg` | float | Origin balance before transaction |
| `newbalanceOrig` | float | Origin balance after transaction |
| `nameDest` | str | Destination account ID (drop for modeling) |
| `oldbalanceDest` | float | Destination balance before transaction |
| `newbalanceDest` | float | Destination balance after transaction |
| `isFraud` | int | **Label.** 1 if fraudulent, else 0 |
| `isFlaggedFraud` | int | **Drop — leaks the label.** Rule-based flag for transfers > 200,000 |

## Caveats for production

PaySim is synthetic, generated by a rule-based simulator. Models trained on it overestimate their real-world performance. The project README explicitly notes this — production deployment would require domain-adapted retraining on real transaction data.
